EPL 2025/26 Match Outcome Prediction Project Draft

Project Goal
Build an EPL match prediction project specifically for the 2025/26 Premier League season. The model should be trained on historical EPL data, while live predictions should use freshly pulled pre-match team context and local caching to avoid unnecessary API calls.

Recommended Scope
- Competition: English Premier League
- Prediction season: 2025/26 only
- Historical training data: prior EPL seasons
- Prediction target: Home Win / Draw / Away Win
- Granularity: one row per match
- Constraint: only use information available before kickoff

Core Research Questions
1. Can pre-match team and fixture information predict EPL match outcomes better than simple baselines?
2. Which features matter most: recent form, goal differential, league ranking, home advantage, rest days, head-to-head, or xG-based measures?
3. How much improvement can a structured ML pipeline deliver over a naive home-win or table-rank baseline?

Primary Data Source
Use football-data.co.uk as the first version data source.

Why this source
- Stable historical EPL coverage
- Easy CSV download
- Includes results and useful match-level fields
- Good for a reproducible student project

Optional Enhanced Sources
- FBref: team and match logs
- Understat: xG and xGA

Version 1 recommendation
Start with football-data.co.uk only. Build a complete reproducible pipeline first. Add FBref or Understat later only if needed.

Target Deliverables
1. Reproducible dataset covering around 10 EPL seasons
2. Clean feature table for pre-match prediction
3. At least two baselines and two ML models
4. Evaluation on season-based holdout data
5. One short written analysis summary
6. Resume-ready project bullet based on measured results

Modeling Plan
Baselines
1. Most frequent class baseline
2. Always predict home win baseline
3. Ranking-difference heuristic baseline

Machine learning models
1. Multinomial Logistic Regression
2. Random Forest
3. Optional: XGBoost or LightGBM if environment setup is easy

Validation Strategy
Do not randomly shuffle all matches together.

Use season-aware splits to avoid leakage.

Recommended split
- Train: 2015-16 to 2021-22
- Validation: 2022-23
- Test: 2023-24 or 2024-25

Better but optional
- Rolling-origin validation across multiple seasons

Evaluation Metrics
Primary metrics
- Accuracy
- Macro F1

Secondary metrics
- Precision by class
- Recall by class
- Confusion matrix
- Calibration or predicted probability quality

Expected Outcome Range
For a three-class EPL outcome model using pre-match features only, a realistic and defensible test accuracy target is roughly 55% to 65%. If the model exceeds 60% on a proper season holdout, that is already resume-worthy.

Avoid
- Reporting extremely high accuracy without strict validation
- Using post-match data in features
- Mixing seasons randomly and calling it test accuracy
